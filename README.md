# 🚗 Malcha (말차) - 자율주행 동키카 학습 데이터 정제 및 관리 시스템

> **"C# WinForm과 WSL을 연동한 대용량 자율주행 학습 데이터 정제 및 관리 플랫폼"**

단순히 주행 데이터를 학습시키는 것을 넘어, 수만 장의 데이터를 시각적으로 모니터링하고 논리적으로 정제하여 모델의 성능을 극대화하기 위해 개발된 데스크톱 애플리케이션입니다. Windows UI와 Linux(WSL) 기반의 딥러닝 환경을 매끄럽게 연결합니다.

## 🛠️ Tech Stack (기술 스택)

### Frontend & UI
* **Language:** C# 10.0+ 
* **Framework:** .NET (Windows Forms)
* **Design Pattern:** Layered Architecture, MVP (Passive View) 기반 설계

### Backend & AI Engine
* **Environment:** Windows Subsystem for Linux (WSL2 - Ubuntu 22.04)
* **Language:** Python 3.9
* **ML Framework:** TensorFlow / Keras (DonkeyCar Framework)

### Data & Communication
* **Storage:** JSON / JSON Lines (`.catalog`, `database.json`)
* **IPC (프로세스 통신):** `System.Diagnostics.Process` 기반 표준 입출력(Standard I/O) 스트림 실시간 파싱

## 📌 1. 프로젝트 배경 및 목표 (Motivation & Goal)

### 프로젝트 배경
* **데이터 관리의 한계:** 자율주행 동키카(DonkeyCar) 플랫폼의 주행 데이터는 수만 줄의 JSON Lines(`.catalog`) 형식과 대규모 이미지 파일로 구성됩니다. 기존 환경에서는 이를 직관적으로 검수할 수 있는 도구가 부족하여, 차량 충돌·이탈·정지 등 모델 학습을 방해하는 '불량 프레임'을 수동으로 찾아내고 정제하기가 극도로 어려웠습니다.
* **CLI 종속성과 이기종 환경의 장벽:** 모델 학습 파이프라인이 리눅스 CLI 환경에 종속되어 있어, 데이터 확인(Windows)과 AI 학습(Linux) 간의 빈번한 콘텍스트 스위칭이 발생했습니다. 텍스트 로그 형태로만 출력되는 학습 진행 상황은 실시간 검증 점수를 한눈에 파악하기 어렵게 만들어 개발 생산성을 저하시켰습니다.

### 프로젝트 목표
* **통합 데이터 정제 플랫폼 구축:** C# Windows Forms 기반의 고속 데이터 시각화 엔진과 마우스 드래그 기반의 주행 데이터 편집 시스템을 결합하여, 누구나 손쉽게 데이터 노이즈를 식별하고 정제할 수 있는 데스크톱 애플리케이션을 개발합니다.
* **Windows-WSL 간 이기종 연동 자동화:** 단일 UI 내부에서 클릭 한 번으로 Windows의 정제 데이터와 Linux(WSL) 하위 시스템의 딥러닝 인프라를 동기화하고, 백그라운드 프로세스 제어를 통해 인공지능 모델 학습 및 검증 과정을 실시간으로 모니터링할 수 있는 심리스(Seamless)한 파이프라인 구축을 목표로 합니다.

## 🏗️ 2. 시스템 설계 및 아키텍처 (Architecture & SE)

### 2.1 핵심 아키텍처 패턴 (MVC 기반 Layered Architecture)
Malcha 프로젝트는 UI(View), 조율자(Controller), 비즈니스 로직(Service/Domain)을 철저히 분리하여 설계되었습니다.
* **Controller**: `CatalogEditorController`, `TrainingController` (화면 제어 및 흐름 통제)
* **Service**: `CatalogService`, `WslDataSyncService` (프레임 병합, 정제, WSL 명령 전달)
* **Model/Data**: 인메모리 스냅샷(Undo), `CatalogMerger` 등 순수 도메인 로직 처리

### 2.2 핵심 유스케이스 (Core Use Cases)
1. **[다중 카탈로그 병합 및 시각화]**: 폴더 형태의 N개의 분할된 데이터(catalog_0, catalog_1)를 `merged_final`이라는 단일 마스터 카탈로그로 병합 후, 차트(조향/쓰로틀)와 타임라인으로 랜더링.
2. **[Time-Machine 스냅샷 백업 및 롤백 복구]**: 필터 정제 시 발생하는 데이터 손실을 막기 위해 가벼운 인메모리(RAM) Undo(`Ctrl+Z`)를 제공하고, 명시적 저장 시 하드디스크에 시간별 백업을 저장. 사용자는 언제든 다이얼로그를 통해 "과거 시점"을 지정하여 즉각 복구 가능.
3. **[WSL 연동 데이터 훈련]**: 윈도우 환경에서 버튼 하나로 WSL 환경의 파이썬 스크립트(`train.py`)를 호출하여 AI 모델을 훈련시키고 실시간으로 Loss 그래프를 확인.

## ✨ 3. 주요 기능 (Key Features)

### 1) 대용량 주행 데이터 로드 및 시간축 통합
* **비동기 스트림 파싱:** `DataManager`가 대규모 JSON Lines 형식의 카탈로그 파일을 `async/await` 기반으로 순차 파싱하여 대용량 파일을 읽을 때도 메인 UI 스레드가 마비(Freezing)되는 현상을 차단했습니다.
* **멀티 카탈로그 자동 병합:** 분할 저장된 여러 주행 데이터 파일들을 입력받았을 때, `CatalogService`가 각 프레임의 고유 유닉스 타임스탬프(`TimestampMs`)를 기준으로 전체 데이터를 시간축 정렬하고 인덱스를 정규화하여 하나의 통합 세션으로 재구성합니다.

### 2) LRU 캐싱 기반 고속 이미지 시각화
* **메모리 보호를 위한 LRU 캐시:** 주행 리스트나 타임라인 스크롤을 초고속으로 이동할 때 프레임당 고해상도 전방 카메라 이미지를 매번 디스크 I/O로 로드하면 메모리 과부하 및 지연이 발생합니다. 이를 해결하기 위해 `CatalogDisplayController` 내부에 LRU(Least Recently Used) 알고리즘 기반 이미지 캐시 메모리를 구현, 고정 크기(Max 100)의 인덱스 버퍼만 유지하여 메모리 누수를 원천 차단하고 부드러운 시각화를 제공합니다.
* **멀티 차트 동기화:** 선택된 프레임의 실제 차량 주행 모드(`Mode`), 조향각(`Angle`), 스로틀(`Throttle`) 수치를 윈폼 차트 컨트롤(`ChartAdapter`)과 라벨에 실시간 연동하여 주행 성향을 직관적으로 실시간 파악할 수 있습니다.

### 3) 드래그 앤 드롭 수동 정제 및 상태 완벽 복구 (Undo)
* **드래그 구간 다중 선택 및 삭제:** 사용자가 타임라인 인터페이스나 프레임 리스트에서 마우스를 드래그하면 `FrameRangeSelection` 구조가 선택 상태를 관리하며, 원하지 않는 주행 구간을 한 번에 '삭제 목록'으로 이동시키는 직관적인 UX를 제공합니다.
* **인덱스 트래킹 Undo/Red 시스템:** 편집 도중 실수가 발생하더라도 `CatalogSession` 내부에 삭제 데이터의 원래 위치 힌트 정보가 포함된 `DeletedFrameEntry` 객체를 스택 상태로 보존합니다. 이를 통해 ' 되돌리기' 실행 시 타 타임라인 데이터와의 충돌 없이 원래 인덱스 위치에 정확하게 삽입/복구하는 완벽한 가상 상태 제어를 지원합니다.

### 4) 알고리즘 기반 노이즈 자동 정제 필터
* **연속 중복 및 절대값 초과 필터:** 센서 오류로 인해 허용 한계치를 벗어난 이상치 데이터(`OutOfRangeLimit`)나, 차량이 정지해 있어 인공지능 모델에게 무의미한 중복 학습을 유발하는 연속 프레임 데이터를 입실론(`ValueEpsilon`) 오차 이내에서 자동 감지하여 제거합니다.
* **단일 프레임 스파이크 제거:** 주행 도중 순간적인 하드웨어 글리치(Glitch)로 인해 조향각이나 스로틀 값이 V자 형태로 급격하게 튀었다가 복귀하는 단일 고립 노이즈(`IsIsolatedSpike`)를 전후 프레임 가속도 변화율 알고리즘을 기반으로 정밀 필터링합니다.
* **논블로킹 진행률 대화상자:** 대량의 데이터에 필터를 일괄 적용할 때 `ProgressDialog`가 백그라운드 작업과 통신하며 실시간 정제율을 표시하고, 사용자 취소 요청 시 `CancellationToken`을 서비스 계층으로 즉각 전파하여 연산을 중단합니다.

### 5) WSL 프로세스 제어 및 실시간 로그 후킹
* **백그라운드 학습 자동화:** 사용자가 지정한 가상 가상 환경 설정을 기반으로 윈도우 프로세스 파이프를 통해 `wsl.exe python train.py`를 원클릭 백그라운드 호출합니다.
* **정규식 기반 실시간 로그 파싱:** 파이썬 엔진이 터미널(stdout/stderr)에 실시간으로 출력하는 Keras 표준 에폭(Epoch) 정보를 `TrainingLogParser`가 정규식 패턴으로 실시간 가로채어 후킹(Hooking)합니다.
* **Loss 기반 검증 점수 환산 및 차트화:** 수집된 원시 Loss 및 Val_Loss 데이터를 `TrainingScore` 알고리즘(`100 / (1 + loss)`)을 통해 100점 만점 형태의 직관적인 점수로 환산하여 메인 화면의 진행 라벨과 훈련 손실 차트(`ChartAdapter`)에 실시간 시각화합니다.

## 🛠️ 4. 트러블슈팅 및 기술적 의사결정 (Trouble Shooting)

### 🚨 Issue 1: 대용량 데이터 필터링 시 UI 프리징 및 I/O 오버헤드 문제
* **원인:** 초기에는 필터를 돌리거나 프레임을 1장 삭제할 때마다 즉시 디스크에 Auto-backup을 남기고 파일을 덮어쓰도록 설계하여 병목(I/O 프리징) 발생.
* **해결 과정:**
  1. 저장 전까지는 순수하게 인메모리(RAM) 스택 자료구조 위주로 스냅샷을 쌓는 시스템(Undo)을 메인으로 사용.
  2. 디스크 백업은 사용자가 명시적으로 '저장' 버튼을 누를 때 1회만 스냅샷을 생성하도록 백업 정책 변경.
* **결과:** 사용자 체감 속도 비약적 향상 및 디스크 수명(SSD 쓰기 양) 보호, 쓸모없는 찌꺼기 백업 파일 양산 방지.

### 🚨 Issue 2: 과거 데이터와 현재 데이터의 혼종(Merge) 발생 문제
* **원인:** 초기 백업 복구 로직(`CatalogMerger`)이 현재 망가진 데이터와 옛날 백업 데이터를 비교하여 빈 자리를 채워 넣는(Merge) 병합 방식으로 동작하여, 돌아가고 싶은 과거 시점 100% 그대로 롤백되지 않음.
* **해결 과정:** 과거 데이터와 현재 데이터를 섞어버리는 `MergeWithBackupAsync` 대신, 사용자가 [복구] 시점을 선택하면 해당 옛날 파일 자체를 `LoadCatalogFileAsync`로 읽어와 메모리에 통째로 '덮어씌우는(Overwrite)' 로직으로 재설계.
* **결과:** 소프트웨어 공학의 Undo/Redo 철학에 맞는 올바르고 안전한 '시점 선택 롤백' 달성.

## 🚀 ?. 빠른 시작 가이드 (Quick Start: 학습 준비 순서)

### 📋 단계별 프로세스

* **1.데이터 선택:** 데이터 선택 버튼을 클릭하여 주행 .catalog 파일을 열기.

* **3.필터 적용:** 필터 적용 버튼을 눌러 중복 프레임, 조향 스파이크, 범위를 초과하는 이상치를 자동으로 정제. (보통 1회 실행으로 충분.)

  (주의: 이 과정을 거치지 않으면 WSL에 남아있는 예전 데이터로 학습이 진행.)

* **5.학습 시작:** 학습 시작 버튼을 눌러 WSL에서 train.py를 실행합니다. (최초 실행 시 mycar 폴더 경로 지정이 필요.)

* 조향(Steering) 및 쓰로틀(Throttle) 그래프를 분석해야 하는 경우 → 차트 영역 확대
& 프레임 목록을 빠르게 탐색해야 하는 경우 → 리스트 영역 확대
