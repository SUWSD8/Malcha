# 🚗 Malcha (말차)

> **DonkeyCar 주행 데이터를 Windows에서 정리하고, WSL에서 AI를 학습시키고, 잘 됐는지 확인하는 데스크톱 프로그램**

<br>

Malcha는 **Windows 화면**과 **WSL(Linux) 학습 환경**을 하나의 앱으로 이어 줍니다.  
터미널을 오가며 명령어를 치지 않아도, 버튼 몇 번으로 **데이터 정제 → 학습 → 교차 테스트**까지 진행할 수 있습니다.

<br>

## 🛠️ 기술 스택

<br>

| 구분 | 사용 기술 |
|------|-----------|
| **앱 (UI)** | C# 10+ · .NET 10 Windows Forms · MVP(Passive View) |
| **AI (학습)** | WSL2 (Ubuntu 22.04) · Python 3.9 · DonkeyCar 5.2 · TensorFlow 2.15.1 / Keras 2.15 |

| **Windows ↔ WSL** | `wsl.exe` 프로세스 + stdout/stderr 실시간 파싱 |

## 🌱 처음 쓰는 분 — 용어 정리

코딩을 몰라도 괜찮습니다. 아래 단어만 알면 대부분의 화면을 이해할 수 있습니다.

<br>

| 용어 | 쉬운 뜻 |
|------|---------|
| **주행 데이터** | 차량 카메라 사진 + 그때의 핸들(조향) · 가속(쓰로틀) 기록 |
| **`.catalog`** | 위 기록이 모여 있는 파일 (JSON Lines 형식) |
| **프레임** | 주행 영상의 한 장면 (사진 1장 + 그 순간의 조향·쓰로틀) |
| **정제** | AI 학습에 방해되는 나쁜 구간·중복·오류 프레임을 빼는 작업 |
| **필터** | 정제를 자동으로 해 주는 기능 (버튼 한 번) |
| **연동** | Windows에서 정리한 데이터를 WSL의 `mycar/data` 폴더로 복사하는 것 |
| **WSL** | Windows 안에서 돌아가는 Linux. DonkeyCar 학습은 여기서 실행됩니다 |
| **학습** | AI가 주행 데이터를 보고 운전법을 배우는 과정 |
| **교차 테스트** | 배운 AI가 “이렇게 조종할 것 같다”고 그린 화살표 vs 실제 기록을 비교하는 기능 |

<br>

> 💡 **도움이 필요하면** 앱 안에서 **F1** 키 또는 **도움말** 버튼을 누르세요.  
> 전체 사용법과 단축키가 정리되어 있습니다.

## ⚡ 이렇게 따라하세요 (6단계)

아래 순서대로 진행하면 됩니다. **4번(연동)** 을 빼먹는 경우가 가장 많으니 주의하세요.

<br>

| # | 버튼 | 무엇을 하나요? |
|:-:|------|----------------|
| **1** | **데이터 선택** | 주행 `.catalog` 파일이 들어 있는 폴더를 엽니다 |
| **2** | (선택) **선택구간 삭제** · **필터 적용** | 손으로 고치거나, 자동 필터로 노이즈·중복을 제거합니다 |
| **3** | **저장** | 지금까지 작업 내용을 디스크에 저장합니다. `backups/` 폴더에 백업도 생깁니다 |
| **4** | **정제 데이터 연동** | ★ 정리한 데이터를 WSL `mycar/data`로 복사합니다. **학습 전에 반드시** |
| **5** | **학습 시작** | 오른쪽 패널에서 WSL 학습을 시작합니다. 처음이면 mycar 폴더 경로를 한 번 지정 |
| **6** | **교차 테스트** | 학습한 모델을 선택하고, 예측(노랑) vs 기록(주황) 화살표를 확인합니다 |

<br>

**2단계 — 수동 vs 필터**

- **선택구간 삭제**: 타임라인이나 목록에서 드래그로 구간을 고른 뒤 삭제합니다. 충돌·이탈 구간처럼 눈으로 보이는 문제에 적합합니다.
- **필터 적용**: 중복 프레임, 조향 급변(스파이크), 범위를 벗어난 값 등을 자동으로 제거합니다. 보통 **「스마트 권장」** 프리셋으로 1회 실행하면 충분합니다.

<br>

**실수했을 때**

- 방금 한 편집 → **되돌리기** (`Ctrl+Z`)
- 예전 시점으로 돌아가기 → **복구** 버튼 (`backups/` 목록에서 선택)
- 모델 되돌리기 → **백업에서 복구** (재학습 전 `.h5` 백업)

## ⭐ 꼭 기억할 3가지

<br>

### 1. 학습 전에 「정제 데이터 연동」은 필수

연동을 하지 않으면, Malcha에서 아무리 데이터를 정리해도 **WSL에 남아 있던 예전 data**로 학습이 진행됩니다.

<br>

### 2. 필터는 「스마트 권장」부터

필터를 너무 강하게 쓰면 주행 중 프레임까지 지워져서, AI가 다양한 장면을 배우지 못합니다.  
**스마트 권장**은 화면이 거의 같을 때만 중복을 제거합니다. 30% 이상 지워지면 경고가 뜹니다.

<br>

### 3. 잘 됐는지는 점수보다 「교차 테스트」

학습 점수(100점 환산)는 참고용입니다.  
**교차 테스트**에서 **노란 화살표(AI 예측)** 가 **주황 화살표(실제 기록)** 를 따라가는지 보는 것이 더 중요합니다.

## 📌 1. 왜 만들었나

<br>

### 배경 — 기존에 불편했던 점

**데이터를 눈으로 확인하기 어려웠습니다.**

DonkeyCar 주행 데이터는 수만 장의 이미지와 `.catalog` 기록으로 구성됩니다.  
충돌·이탈·정지 같은 **나쁜 프레임**을 찾으려면 파일을 직접 뒤져야 했고, GUI 도구가 없어 검수가 매우 힘들었습니다.

<br>

**Windows와 Linux를 오가야 했습니다.**

데이터 확인은 Windows에서, AI 학습은 Linux CLI에서 진행됩니다.  
학습 진행 상황도 터미널 로그로만 볼 수 있어, 한눈에 파악하기 어려웠습니다.

<br>

### 목표 — Malcha가 해결하려는 것

1. **Windows에서 데이터를 보고, 고치고, 저장**할 수 있는 편집 도구
2. **버튼 한 번으로 WSL과 동기화**하고, 학습·검증까지 같은 화면에서 진행
3. **실수해도 되돌릴 수 있는** 백업·복구·모델 보호

## 🏗️ 2. 프로그램 구조

<br>

Malcha는 **화면(View) · 흐름(Controller) · 실제 작업(Service)** 을 분리해서 만들었습니다.  
한 파일에 모든 코드를 넣지 않아, 기능 추가·수정이 수월합니다.

<br>

```
Form1 (화면)
    ↓
Controller (편집 · 학습 · 교차테스트 흐름)
    ↓
Service (WSL 실행 · 데이터 연동 · 로그 파싱)
    ↓
Model / Data (카탈로그 · 세션 · 필터 로직)
```

<br>

| 계층 | 하는 일 | 대표 파일 |
|------|---------|-----------|
| **Controller** | 버튼 클릭 → 어떤 순서로 처리할지 결정 | `CatalogEditorController`, `TrainingController`, `CrossTestController` |
| **Service** | WSL 명령 실행, 데이터 복사, 학습 로그 읽기 | `WslDataSyncService`, `WslTrainingService`, `WslCrossTestService` |
| **Model / Data** | 카탈로그 읽기·병합, Undo, 필터 알고리즘 | `CatalogSession`, `FrameRefinementFilter`, `CatalogMerger` |

<br>

### 핵심 흐름 3가지

<br>

**① 여러 catalog → 하나로 합치기**

`catalog_0`, `catalog_1` … 처럼 나뉜 파일을 시간순으로 정렬해 `merged_final` 하나로 병합합니다.

<br>

**② 편집 중 실수 → 되돌리기 / 복구**

- 편집 중: 메모리(RAM)에 Undo 스택 → `Ctrl+Z`로 즉시 되돌리기
- **저장** 버튼: 디스크에 저장 + `backups/`에 시점별 백업
- **복구** 버튼: 원하는 백업 시점을 통째로 불러오기

<br>

**③ WSL 학습 + 실시간 모니터링**

Windows에서 `wsl.exe python train.py`를 실행하고, 터미널에 찍히는 epoch·Loss 정보를 읽어 화면에 표시합니다.

## ✨ 3. 주요 기능

<br>

### 데이터 불러오기 · 병합

- 대용량 `.catalog` 파일을 **비동기**로 읽어 UI가 멈추지 않습니다
- 여러 catalog를 **타임스탬프 기준**으로 정렬·병합합니다

<br>

### 이미지 · 차트 표시

- 자주 보는 이미지는 **캐시**에 두어 스크롤·재생이 부드럽습니다
- 선택한 프레임의 **조향·쓰로틀·주행 모드**를 차트와 라벨에 동시에 표시합니다

<br>

### 수동 정제 (드래그 · 구간 삭제)

- 타임라인이나 목록에서 **드래그**로 구간 선택
- 선택 구간을 **삭제 목록**으로 보내거나, **되돌리기**로 복원
- 삭제 시 **원래 위치 정보**를 기억해, Undo 시 정확히 되살립니다

<br>

### 자동 필터 (노이즈 제거)

| 필터 종류 | 무엇을 제거하나요? |
|-----------|-------------------|
| **범위 초과** | 조향·쓰로틀이 허용 범위를 벗어난 값 |
| **연속 중복** | 차가 거의 멈춰 있고, 값도 거의 같은 연속 프레임 |
| **스파이크** | 한 프레임만 조향·쓰로틀이 V자로 튀었다가 돌아오는 글리치 |
| **스마트 이미지** | 화면이 비슷하고 (정지 또는 저속)일 때만 중복 제거 — **주행 장면은 유지** |

<br>

필터 실행 중에는 **진행률 창**이 뜨고, 취소하면 중간에 멈출 수 있습니다.

<br>

### WSL 학습 · 모니터링

- **학습 시작** 한 번으로 WSL `train.py` 실행
- epoch·Loss 로그를 실시간으로 읽어 **100점 만점 점수**와 그래프로 표시
- **강제 종료** 시 상황에 따라 epoch까지 저장하거나, 기존 모델을 보호

<br>

### 모델 보호 (재학습·강제 종료)

| 상황 | 동작 |
|------|------|
| **재학습 시작** | 기존 `mypilot.h5` → `mypilot.malcha_backup.h5`로 백업 |
| **학습 중** | `mypilot.malcha_staging.h5`에만 저장 (완료 전까지 기존 모델 유지) |
| **학습 완료** | staging → `mypilot.h5`로 반영 |
| **되돌리기** | **백업에서 복구** 버튼으로 `.malcha_backup.h5` 복원 |

<br>

### 교차 테스트

- 학습한 모델로 **예측 조향·쓰로틀**을 계산
- 영상 위에 **주황(실제)** · **노랑(예측)** 화살표를 겹쳐 표시
- In ~ Out 구간만 골라 테스트 가능

<br>

### 최근 추가·개선 기능 요약

| 기능 | 한 줄 설명 |
|------|-----------|
| **스마트 필터** | 화면이 거의 같을 때만 중복 제거 |
| **필터 프리셋** | 교차테스트 / 스마트 권장 / 약함 / 강함 / 직접 설정 |
| **교차 테스트** | AI 예측 vs 실제 기록 화살표 비교 |
| **모델 staging** | 학습 완료 전까지 기존 `.h5` 덮어쓰지 않음 |
| **백업에서 복구** | 재학습 전 모델을 버튼으로 되돌리기 |
| **연동 지문(fingerprint)** | 정제 후 연동을 안 했으면 학습 전 경고 |

## 🛠️ 4. 개발 중 고친 문제

<br>

### Issue 1 — 편집할 때마다 느려짐

| | |
|---|---|
| **원인** | 프레임 하나 지울 때마다 디스크에 백업을 남겨 I/O 병목 발생 |
| **해결** | 편집 중에는 **메모리 Undo**만 사용. **저장** 버튼을 눌렀을 때만 `backups/` 생성 |
| **결과** | 체감 속도 개선, 불필요한 백업 파일 감소 |

<br>

### Issue 2 — 복구해도 데이터가 섞임

| | |
|---|---|
| **원인** | 예전 백업과 현재 데이터를 **merge(병합)** 해서, 원하는 시점으로 100% 돌아가지 않음 |
| **해결** | **복구** 시 선택한 백업 파일을 **통째로 덮어쓰기** |
| **결과** | “이 시점으로 돌아가기”가 정확하게 동작 |

<br>

### Issue 3 — 필터가 너무 많이 지움

| | |
|---|---|
| **원인** | 조향·쓰로틀 숫자만 같으면 주행 중 프레임까지 삭제 → AI 학습 품질 저하 |
| **해결** | **스마트 이미지 필터** — 화면이 비슷하고 정지/저속일 때만 제거. 30% 이상 제거 시 경고 |
| **결과** | 교차 테스트에서 예측 품질 유지 |

<br>

### Issue 4 — 재학습·강제 종료 시 모델 파괴

| | |
|---|---|
| **원인** | `train.py`가 epoch 1부터 `mypilot.h5`에 바로 저장 → 기존 모델 덮어씀 |
| **해결** | **staging** 임시 파일에 학습 → 완료 시만 반영 + `.malcha_backup.h5` + **백업에서 복구** 버튼 |
| **결과** | 재학습·중단해도 기존 모델 보호 |
